## 2. P1-1 节点持久化任务提示词

请完成 Treeify P1-1：节点编辑、删除、执行状态持久化。

背景：

- 当前前端可以编辑节点、删除节点、切换执行状态，但刷新后可能丢失。
- 后端已有相关接口：
  - `PUT /api/v1/cases/{caseId}`
  - `DELETE /api/v1/cases/{caseId}`
  - `PATCH /api/v1/cases/{caseId}/execution-status`
  - `PUT /api/v1/projects/{projectId}/mindmap`
- 脑图保存当前可作为兜底，全量保存节点布局。

重点阅读文件：

- `frontend/src/features/workspace/workspaceStore.ts`
- `frontend/src/features/workspace/useMindmapSave.ts`
- `frontend/src/components/NodeCard.tsx`
- `frontend/src/components/SelectionBar.tsx`
- `frontend/src/components/Toolbar.tsx`
- `frontend/src/shared/api/treeify.ts`
- `src/main/java/com/zoujuexian/aiagentdemo/api/controller/treeify/TestCaseController.java`
- `src/main/java/com/zoujuexian/aiagentdemo/service/treeify/TreeifyPersistenceService.java`

目标：

1. 编辑节点标题、优先级、标签后，调用后端保存，刷新仍保留。
2. 删除节点后，调用后端删除，刷新不恢复。
3. 切换执行状态后，调用后端状态接口，刷新仍保留。
4. 保留脑图全量保存作为布局兜底。
5. 请求失败时给出明确错误状态，不要静默吞掉。

验收标准：

- 编辑节点标题后刷新，标题保持修改后的值。
- 切换执行状态为 `passed` 后刷新，状态保持 `passed`。
- 删除一个节点后刷新，节点不再出现。
- 前端 mock 模式下仍可本地演示，不因真实接口缺失而崩溃。
- TypeScript 构建通过，后端已有接口不被破坏。

请直接实现，并在完成后列出修改文件和验证命令结果。

## 4. P1-3 Step 中间结果传递任务提示词

请完成 Treeify P1-3：Step 模式 E1/E2/E3 中间结果传递。

背景：

- Auto 模式下 E1、E2、E3 是连续执行的，后续阶段可以使用前阶段结果。
- Step 模式下当前 confirm 后，E2/E3 主要基于原始 input 重新生成，未充分使用上一阶段结果。
- 这会降低 Step 模式生成质量。

重点阅读文件：

- `src/main/java/com/zoujuexian/aiagentdemo/service/treeify/AiTreeifyGenerationService.java`
- `src/main/java/com/zoujuexian/aiagentdemo/service/treeify/MockTreeifyService.java`
- `src/main/java/com/zoujuexian/aiagentdemo/service/treeify/TreeifyPersistenceService.java`
- `src/main/java/com/zoujuexian/aiagentdemo/domain/entity/TreeifyGenerationTask.java`
- `src/main/java/com/zoujuexian/aiagentdemo/api/controller/treeify/GenerateController.java`
- `frontend/src/features/generation/useGenerateStream.ts`
- `frontend/src/features/generation/generationStore.ts`

目标：

1. Step 模式 E1 完成后保存 E1 结构化结果。
2. 用户确认 E1 后，E2 prompt 使用原始需求 + E1 结果。
3. Step 模式 E2 完成后保存 E2 结构化结果。
4. 用户确认 E2 后，E3 prompt 使用原始需求 + E1 结果 + E2 结果。
5. 可选支持 confirm 时传入 `feedback`，作为下一阶段补充约束。
6. Auto 模式行为不受影响。

验收标准：

- Step 模式 E1 `stage_done` 后任务进入 `wait_confirm`。
- confirm E1 后，E2 生成内容明显基于 E1 结果。
- confirm E2 后，E3 生成内容明显基于 E1/E2 结果。
- 任务刷新和重新订阅 SSE 时状态保持正确。
- mock fallback 行为不被破坏。
- `./mvnw -q test` 或至少 `./mvnw -q -DskipTests compile` 通过。

请直接实现，并在完成后列出修改文件和验证命令结果。
