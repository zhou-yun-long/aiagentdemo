下面这版可以直接放进简历：

**Treeify：AI 驱动测试用例生成平台**  
**技术栈：Spring Boot、Spring AI、React、TypeScript、Vite、MySQL、PostgreSQL/pgvector、Redis、Docker、MCP、SSE**

负责设计并实现 AI 驱动的测试用例生成平台，面向需求文档、历史用例和测试规范，自动完成需求理解、测试对象拆解、测试场景生成与质量评审，提升测试用例设计效率和覆盖完整性。

- 基于 Spring Boot + Spring AI 构建 AI Agent 编排层，设计 E1 需求分析、E2 测试对象拆解、E3 场景生成三阶段 Sub-agent 流程，支持自动模式与人工确认的逐步模式。
- 设计 RAG 知识库能力，结合 pgvector 向量检索、全文检索与 Top-K 召回，将项目摘要、历史用例、规范文档注入 Prompt，提升生成结果与业务背景的一致性。
- 实现 Critic Agent 质量评审机制，从测试对象覆盖率、路径完整性、可执行性、重复率等维度评分，对低质量结果触发重试优化。
- 设计 SSE 流式生成接口与前端实时渲染流程，支持阶段进度展示、流式内容输出、用例预览、内联编辑与批量确认。
- 规划项目管理、摘要 Agent、用例管理、知识库管理、MCP 外部工具集成等核心模块，支持与 Jira、禅道、CI 等工具扩展对接。
- 设计 MySQL + PostgreSQL/pgvector 双库架构，业务数据与向量数据通过 project_id 关联，兼顾业务查询、语义检索和后续扩展能力。

**精简版：**

基于 Spring Boot、Spring AI、React 和 pgvector 设计 AI 驱动测试用例生成平台，通过三阶段 Sub-agent 编排完成需求分析、测试对象拆解和用例生成；结合 RAG 检索历史用例与规范文档，使用 Critic Agent 对生成结果进行覆盖率、可执行性和重复率评审，并通过 SSE 实现前端流式展示与逐步确认交互，提升测试用例生成效率和业务一致性。
